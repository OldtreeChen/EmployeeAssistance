# LINE 打卡系統 — 部署說明

## 系統架構

```
LINE App
  │  (Webhook / LIFF)
  ▼
Nginx (443 HTTPS)  ← AWS EC2 VM
  │
  ▼
Node.js App (port 3000)
  │
  ▼
MySQL 8.0
```

---

## 前置準備

### 1. LINE Developers Console 設定

1. 前往 https://developers.line.biz/console/
2. 選擇您的 Provider → **建立 Messaging API Channel**（若已有請跳過）
3. 記下以下資訊：
   - **Channel Secret**
   - **Channel Access Token**（長期）

4. 建立 **LIFF App**：
   - 進入 Channel → **LIFF** 分頁 → Add
   - Size：`Full`
   - Endpoint URL：`https://your-domain.com/liff`（先填，之後更新）
   - 記下 **LIFF ID**（格式：`1234567890-xxxxxxxx`）

---

### 2. AWS EC2 準備

```bash
# 安裝 Docker & Docker Compose（Ubuntu 22.04）
sudo apt update
sudo apt install -y docker.io docker-compose-plugin
sudo systemctl enable --now docker
sudo usermod -aG docker $USER
```

**Security Group 開放：**
- TCP 22   (SSH)
- TCP 80   (HTTP)
- TCP 443  (HTTPS)

---

### 3. SSL 憑證（使用 Let's Encrypt）

```bash
sudo apt install -y certbot
sudo certbot certonly --standalone -d your-domain.com

# 憑證路徑
# /etc/letsencrypt/live/your-domain.com/fullchain.pem
# /etc/letsencrypt/live/your-domain.com/privkey.pem
```

## 部署步驟

### Step 1：上傳程式碼

```bash
# 在 EC2 上
git clone https://github.com/your-org/line-punch-system.git
cd line-punch-system
```

或直接用 scp 上傳：

```bash
scp -r ./line-punch-system ec2-user@your-ec2-ip:~/
```

---

### Step 2：設定環境變數

```bash
cp .env.example .env
nano .env
```

填入以下內容：

```
LINE_CHANNEL_ACCESS_TOKEN=（從 LINE Developers Console 取得）
LINE_CHANNEL_SECRET=（從 LINE Developers Console 取得）
LINE_LIFF_ID=（LIFF App ID）

DB_HOST=db
DB_PORT=3306
DB_USER=punchuser
DB_PASSWORD=（設定一個強密碼）
DB_NAME=line_punch_system
DB_ROOT_PASSWORD=（設定 root 密碼）
```

---

### Step 3：設定 SSL 憑證

```bash
mkdir -p nginx/certs
sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem nginx/certs/
sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem  nginx/certs/
sudo chown $USER:$USER nginx/certs/*
```

---

### Step 4：更新 Nginx 設定

編輯 `nginx/nginx.conf`，將 `your-domain.com` 替換為您的實際網域。

---

### Step 5：準備 Rich Menu 圖片

```bash
# 安裝 Inkscape（SVG 轉 PNG）
sudo apt install -y inkscape

# 轉換（需 2500x843 px，PNG 格式）
inkscape scripts/rich-menu.svg \
  --export-type=png \
  --export-filename=scripts/rich-menu.png \
  --export-width=2500 \
  --export-height=843
```

---

### Step 6：啟動所有服務

```bash
docker compose up -d --build
```

確認服務狀態：

```bash
docker compose ps
docker compose logs -f app
```

---

### Step 7：更新 LIFF Endpoint URL

回到 LINE Developers Console → LIFF → 將 Endpoint URL 改為：

```
https://your-domain.com/liff
```

---

### Step 8：設定 Webhook URL

LINE Developers Console → Messaging API → Webhook URL：

```
https://your-domain.com/webhook
```

點擊 **Verify** 確認連線成功（應顯示 Success）。

---

### Step 9：建立 Rich Menu

```bash
# 進入 app container 執行腳本
docker compose exec app node scripts/setup-rich-menu.js
```

---

### Step 10：測試

1. 用 LINE 掃描 Official Account 的 QR Code 加入好友
2. 確認下方出現「📋 打卡 / 工時」選單
3. 點擊「上班打卡」→ 應收到確認訊息
4. 點擊「填寫工時」→ 應開啟工時表單

## 常用指令

```bash
# 查看 log
docker compose logs -f app

# 重新建置
docker compose up -d --build app

# 進入 MySQL
docker compose exec db mysql -u punchuser -p line_punch_system

# 查詢今日打卡
SELECT u.display_name, a.punch_type, a.punch_time
FROM attendance a
JOIN users u ON a.line_user_id = u.line_user_id
WHERE DATE(CONVERT_TZ(a.punch_time,'+00:00','+08:00')) = CURDATE()
ORDER BY a.punch_time;

# 更新 Rich Menu（修改後重跑）
docker compose exec app node scripts/setup-rich-menu.js
```

## SSL 憑證自動更新

```bash
# 加入 crontab（每月更新）
echo "0 3 1 * * certbot renew --quiet && \
  cp /etc/letsencrypt/live/your-domain.com/fullchain.pem ~/line-punch-system/nginx/certs/ && \
  cp /etc/letsencrypt/live/your-domain.com/privkey.pem  ~/line-punch-system/nginx/certs/ && \
  cd ~/line-punch-system && docker compose restart nginx" | crontab -
```

## 遇到問題？

| 問題 | 解法 |
|------|------|
| Webhook Verify 失敗 | 確認 443 port 已開放、SSL 憑證正確 |
| 打卡無回應 | `docker compose logs app` 查看錯誤 |
| LIFF 無法開啟 | 確認 LIFF Endpoint URL 已更新並為 HTTPS |
| 資料庫連線失敗 | 確認 .env DB 設定正確，`docker compose ps db` 確認健康 |
